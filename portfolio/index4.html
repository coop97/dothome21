사이트를 제작하기 전에 UI/UX 환경분석으로 기본 개념과 최근 트랜드를 분석하고, 직접 콘셉트를 기획하여 사용자의 요구사항을 파악하며 웹사이트에 대한 이해도를 높였습니다. HTML과 CSS에 대한 기본 지식만 있으면 레이아웃을 작성하여 웹사이트를 제작할 수 있습니다. 기존의 웹사이트를 따라 만들면서 사이트의 기본 구조를 파악하고, 외형만이 아니라 기능
구현까지 범위를 확장했습니다. 클론코딩으로 연습을 마친 뒤, HTML과 CSS에서 나아가 SCSS, JAVASCRIPT, JQUERY, PHP, REACT와 VIEW를 사용하여 새로운 웹사이트를 만들었습니다. 웹사이트를 만들 때는 사용자의 편의를 고려하여 깔끔한 외형과 편리한 기능을 추가하고, 다양한 기기의 화면에 맞춰 반응형으로 작업했습니다.

자바스크립트는 동적인 화면을 웹페이지에 구현하기 위해 사용하는 객체지향 스크립트 또는 프로그래밍 언어입니다. 또한 컴파일러를 거치지 않고 실행시키는 인터프리트 언어입니다. 자바스크립트는 문서 객체 모델로, 브라우저 간 호환성이 떨어져 크로스 브라우징이 어렵다는 단점이 있습니다. 제이쿼리는 자바스크립트의 생산성을 향상시켜주는 자바스크립트 라이브러
리로, 이러한 자바스크립트의 단점을 보완합니다. 스크립트는 총 4개로 분류하여 같은 스크립트를 자바스크립트와 제이쿼리 버전으로 만들었습니다. 마우스의 움직임를 이용한 마우스 효과, 스크롤링을 이용한 패럴랙스 효과, 이미지를 이용한 슬라이더 효과 그리고 다양한 기능을 종합하고 규칙을 정한 게임 효과가 있습니다.

animation

애니메이션은 사이트에 생동감을 부여하고, 종류가 다양합니다. CSS 애니메이션을 사용하면 간단한 애니메이션을 만들 수 있습니다. 여기에 자바스크립트를 추가하면 CSS 애니메이션을 조정할 수 있고, 짧은 코드로 더 효과적인 애니메이션을 만들 수 있습니다. CSS 트랜지션은 CSS 속성을 변경할 때 애니메이션 속도를 조절할 때 사용합니다. CSS 문법인 @keyframes을
사용하면 간단한 애니메이션 여러 개를 한꺼번에 실행시킬 수 있습니다. 애니메이션은 온라인 코드 편집기이자 커뮤니티인 CODEPEN에서 제작했습니다. HTML과 CSS 뿐만 아니라 템플릿 엔진의 종류 중 하나이면서 HTML의 가독성을 높여주는 PUG와 Sass의 3버전으로 CSS 구문과 완전히 호환되는 SCSS를 사용했습니다.
